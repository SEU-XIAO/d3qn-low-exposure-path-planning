# 基于三维栅格战场环境的低暴露路径规划项目技术路线

## 1. 项目目标

本项目拟构建一个三维军事模拟栅格环境，在存在敌方视线威胁和障碍物遮蔽的条件下，研究智能体从起点到终点的低风险路径规划问题。

项目的核心目标不是单纯寻找最短路径，而是在以下多个目标之间取得平衡：

- 尽量到达目标点
- 尽量缩短路径长度
- 尽量降低暴露在敌方视线中的风险
- 尽量利用障碍物形成掩体
- 在不同场景中具备一定泛化能力

因此，本项目本质上是一个“受威胁约束的三维路径规划 + 深度强化学习决策”问题。

## 2. 问题重构与研究定义

### 2.1 环境定义

环境定义为离散化的三维栅格空间：

- 地图底面为二维正方形区域
- 每个网格单元具有高度属性
- 障碍物占据若干连续栅格，并具有不同高度、长度与形状
- 智能体从起点出发，目标是到达终点
- 敌人位于地图边界某一点或某一段区域
- 敌人具有方向性视场，形成一个带角度约束和距离约束的威胁区域

为保证问题可计算、可训练，建议将原始连续三维空间简化为“二维平面移动 + 高度遮挡判定”的建模方式。也就是说：

- 智能体主要在地面网格上移动
- 障碍物高度决定是否遮挡敌人视线
- 风险计算通过“敌我连线是否被障碍物遮挡”来完成

这种建模方式比直接做全三维自由运动更合理，既保留了军事掩体利用的关键特征，又显著降低了状态空间和训练难度。

### 2.2 任务定义

给定：

- 三维栅格地图
- 起点与终点
- 障碍物分布
- 敌方位置、朝向、视场角和探测半径

输出：

- 一条从起点到终点的移动策略或路径

优化目标：

- 路径总代价最小
- 暴露风险最小
- 到达成功率最高

## 3. 总体技术路线

建议采用“环境建模 + 风险场构建 + 强化学习决策 + 基线算法对比”的技术路线。

整体流程如下：

1. 构建三维栅格战场环境
2. 建立敌方视线与遮挡计算模型
3. 将环境转化为强化学习可交互任务
4. 设计状态、动作、奖励函数
5. 训练路径规划智能体
6. 与传统路径规划算法进行对比实验
7. 评估模型在不同地图上的泛化效果

这一路线的优势在于：

- 有明确的工程落地顺序
- 能形成可对比、可解释的实验结果
- 后续可以自然扩展到多敌人、多目标和动态敌情场景

## 4. 环境建模方案

### 4.1 栅格尺度设计

栅格尺度不宜过细，也不宜过粗。建议优先采用以下离散方案：

- 平面尺寸：`32 x 32` 或 `64 x 64`
- 高度层数：`8` 到 `16`
- 单个网格代表一个基础空间单元

推荐原因：

- `32 x 32` 适合作为算法验证与快速训练版本
- `64 x 64` 更适合作为正式实验版本
- 高度层数控制在 `8-16` 有利于表达遮挡关系，同时避免计算量过大

建议先做两阶段设计：

- 第一阶段：`32 x 32 x 8`
- 第二阶段：`64 x 64 x 16`

第一阶段用于验证环境、奖励函数和算法可行性；第二阶段用于提升环境复杂度和结果可信度。

### 4.2 障碍物设计

障碍物不仅决定可通行性，也决定遮挡效果，因此应具备几何多样性。

建议设置以下几类障碍物：

- 立方体障碍物：用于模拟建筑、掩体箱体
- 长条形障碍物：用于模拟围墙、壕沟边缘、连续掩体
- 不规则组合障碍物：由多个立方体拼接形成

建议控制参数包括：

- 高度：低、中、高三档
- 长度：短条、中条、长条
- 宽度：1 格或多格
- 分布方式：随机分布 + 规则分布混合

建议约束：

- 不能完全堵死起点和终点
- 至少存在一条可行路径
- 障碍物密度控制在合理范围内，例如 `10% - 25%`

### 4.3 敌方视线与威胁建模

敌方威胁不建议仅定义为简单的几何扇形，而应定义为“方向约束 + 距离衰减 + 遮挡判定”的风险模型。

建议敌方感知模型包含以下参数：

- 敌人位置 `(x, y, z)`
- 朝向向量
- 最大探测距离
- 水平视场角
- 可选的垂直视场角

风险判定流程建议如下：

1. 判断目标网格是否位于敌方视场角内
2. 判断目标网格是否在最大探测距离内
3. 进行敌人与目标点之间的视线连通检测
4. 若中间存在高于视线的障碍物，则视为被遮挡
5. 若无遮挡，则按距离和角度计算风险值

建议将风险值定义为连续量，而不是简单的“看见/看不见”二值量。这样更适合强化学习优化。

例如可将单点风险定义为：

`Risk = alpha * 距离风险 + beta * 角度风险 + gamma * 暴露持续惩罚`

## 5. 强化学习任务建模

### 5.1 状态空间设计

状态设计应尽量同时包含局部可学习信息和全局目标信息。

建议状态由以下部分组成：

- 当前智能体位置
- 终点相对位置
- 周围局部栅格观测窗口
- 局部障碍物高度信息
- 局部风险热力图
- 敌方相对方向与距离

建议采用两种状态表达方案逐步推进：

#### 方案 A：结构化低维状态

包括：

- 智能体坐标
- 终点坐标
- 最近若干方向上的障碍距离
- 当前风险值
- 与敌人夹角、距离

优点：

- 实现简单
- 训练更稳定
- 适合前期验证

#### 方案 B：局部张量输入

将智能体附近裁剪为固定窗口，如 `11 x 11 x H`，输入多个通道：

- 占用通道
- 高度通道
- 风险通道
- 目标方向通道

优点：

- 更适合卷积网络提取空间结构
- 更容易学习掩体利用模式
- 泛化能力通常优于手工特征

建议项目采用“先 A 后 B”的推进方式。

### 5.2 动作空间设计

建议先采用离散动作空间，降低训练难度。

可选动作包括：

- 上
- 下
- 左
- 右
- 左上
- 左下
- 右上
- 右下
- 原地等待（可选）

如果后期需要考虑更强的战术性，还可以增加：

- 蹲伏/隐蔽状态切换
- 不同速度移动
- 探测动作

但在初版中不建议动作空间过大，否则训练会明显变难。

### 5.3 奖励函数设计

奖励函数是本项目成败的关键，建议采用多目标加权设计。

可定义单步奖励为：

`Reward = 到达奖励 - 步长惩罚 - 风险惩罚 - 非法动作惩罚 + 接近目标奖励`

建议组成如下：

- 到达终点：大额正奖励
- 每走一步：小额负奖励，鼓励短路径
- 暴露在敌方视线内：负奖励
- 暴露风险越高：惩罚越大
- 撞障碍或越界：额外负奖励
- 朝目标接近：小额正奖励
- 长时间停滞：额外负奖励

需要注意：

- 风险惩罚不能过大，否则智能体可能学会“躲着不动”
- 路径惩罚不能过大，否则智能体可能冒高风险走最短线
- 建议通过消融实验调参，而不是一次性固定

## 6. 模型与算法选型

### 6.1 推荐算法路线

对于本项目，推荐采用分阶段算法路线，而不是一开始就直接上复杂模型。

#### 第一阶段：传统算法建立基线

先实现以下基线方法：

- A*
- Dijkstra
- Risk-A*（在路径代价中加入风险项）

作用：

- 验证环境建模是否合理
- 形成可解释的性能下界和上界
- 为强化学习结果提供对照组

#### 第二阶段：值函数型强化学习

建议优先选用：

- DQN
- Double DQN
- Dueling DQN

适用原因：

- 动作空间是离散的
- 工程实现成熟
- 更适合初期调试

其中最推荐的是 `Double DQN + Dueling Network` 组合，稳定性和效果通常优于基础 DQN。

#### 第三阶段：策略梯度或 Actor-Critic

如果后续环境更复杂，可扩展到：

- PPO
- A2C / A3C

适用场景：

- 状态更复杂
- 需要更稳的训练过程
- 未来可能引入连续动作或多智能体扩展

### 6.2 神经网络结构建议

网络结构应与状态表达匹配。

#### 若使用结构化低维状态

建议采用多层感知机（MLP）：

- 输入层：状态特征向量
- 隐藏层：2 到 4 层全连接
- 激活函数：ReLU
- 输出层：各离散动作 Q 值

推荐隐藏层规模：

- `128 -> 128`
- 或 `256 -> 256`

#### 若使用局部栅格张量输入

建议采用卷积网络（CNN）：

- 输入：多通道局部地图张量
- 卷积层提取空间特征
- 全连接层输出动作价值或策略分布

推荐结构示意：

1. `Conv`
2. `Conv`
3. `Flatten`
4. `FC`
5. `Output`

若需要同时融合全局信息，可采用“双分支结构”：

- 分支一：CNN 处理局部地图
- 分支二：MLP 处理全局目标和敌方参数
- 最后进行特征拼接

这是比单纯堆叠网络更合理的方案。

## 7. 训练流程设计

### 7.1 训练阶段划分

建议分三步训练：

#### 阶段一：简化环境预训练

- 固定单个敌人
- 障碍物数量较少
- 起点终点固定或半随机
- 目标是先学会避障和到达

#### 阶段二：风险规避训练

- 引入完整敌方视场
- 增加障碍物形状复杂度
- 起点终点随机化
- 目标是学会利用掩体绕行

#### 阶段三：泛化训练

- 地图随机生成
- 障碍物密度变化
- 敌人位置和朝向随机
- 评估不同场景下稳定性

### 7.2 关键训练机制

建议加入以下机制提升训练效果：

- Experience Replay
- Target Network
- Epsilon-Greedy 探索
- Curriculum Learning
- Reward Normalization

其中，课程学习尤其重要。建议让任务复杂度逐步提升，而不是从高难环境直接开始训练。

## 8. 评价指标与实验设计

为了证明技术路线有效，实验部分不能只看是否到达终点，建议至少包含以下指标：

- 成功到达率
- 平均路径长度
- 平均累计风险
- 平均暴露步数
- 平均决策步数
- 不同地图上的泛化表现

建议设计以下对比实验：

### 8.1 算法对比

- A*
- Risk-A*
- DQN
- Double DQN
- PPO（可选）

### 8.2 消融实验

- 去掉风险项
- 去掉遮挡判定
- 去掉高度信息
- 去掉局部地图输入

### 8.3 泛化实验

- 在训练地图测试
- 在未见过的新地图测试
- 在不同障碍密度下测试
- 在不同敌方朝向下测试

## 9. 工程实现建议

### 9.1 开发栈建议

推荐技术栈如下：

- Python
- NumPy
- PyTorch
- Gymnasium 或自定义 RL 环境接口
- Matplotlib / Plotly 用于可视化

如果希望更规范，环境层可以按如下方式拆分：

- `env/`：地图、敌人、障碍物、风险场
- `agent/`：强化学习算法实现
- `planner/`：A* 等传统算法基线
- `train/`：训练脚本
- `eval/`：实验评估脚本
- `visualize/`：路径与风险热力图展示

### 9.2 可视化建议

建议至少实现以下可视化输出：

- 地图俯视图
- 障碍物高度图
- 敌方视场范围图
- 风险热力图
- 智能体轨迹图

如果条件允许，可以进一步做三维动画演示，提高项目展示效果。

## 10. 可能风险与应对策略

### 10.1 训练不收敛

原因可能包括：

- 奖励设计不合理
- 状态表达不足
- 环境过难

应对方法：

- 先做小地图
- 先做低维状态
- 使用课程学习
- 先用基线算法验证环境

### 10.2 智能体只学会走最短路，不学会规避风险

说明风险惩罚权重偏低，应提升暴露代价，并统计累计风险而非单步风险。

### 10.3 智能体只会躲避，不前进

说明风险惩罚过强或到达奖励不足，应重新平衡奖励系数。

## 11. 推荐实施路线

为了提高项目成功率，建议按以下顺序推进：

### 第一阶段：问题落地

- 完成三维栅格环境定义
- 完成障碍物生成
- 完成敌方视场与遮挡计算
- 完成风险热力图生成

### 第二阶段：建立可解释基线

- 实现 A*
- 实现带风险代价的 Risk-A*
- 输出基线路径和风险对比图

### 第三阶段：强化学习验证

- 先用低维状态 + DQN 跑通
- 再升级到 Double DQN / Dueling DQN
- 最后尝试 CNN 输入的局部地图状态

### 第四阶段：实验与总结

- 完成对比实验
- 完成消融实验
- 完成可视化展示
- 总结模型优势与局限

## 12. 最终建议

从项目可行性、实现难度和结果质量来看，最合理的初版路线不是直接追求复杂三维连续运动，而是采用以下方案：

- 用三维栅格表达地形和障碍高度
- 用二维移动表达智能体行动
- 用“视场角 + 探测距离 + 遮挡检测”表达敌方威胁
- 用 Risk-A* 作为传统基线
- 用 Double DQN / Dueling DQN 作为强化学习主模型
- 用课程学习逐步提升场景复杂度

该路线兼顾了研究价值、实现成本和实验可验证性，适合作为一个完整的项目方案继续向下推进。
